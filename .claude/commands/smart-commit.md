# 変更ごとの自動コミットルール

現在のワークスペースで変更されたファイル（ステージング、未ステージングの両方）を確認し、以下の手順に従ってコミットを行ってください。

## 実行手順

1. `git status` および `git diff` を確認し、どのような変更が行われたかを分析してください。
2. 変更が複数の異なる目的（例：バグ修正とリファクタリングなど）に分かれている場合は、必ず**変更の目的（論理的単位）ごとにファイルを分けてステージング（git add）**してください。
3. 分割した単位ごとに、適切なコミットメッセージを作成してコミットを実行してください。
4. すべての変更がコミットされるまで、このプロセスを繰り返してください。

## コミットメッセージの規約 (Conventional Commits)

メッセージは以下のフォーマット（プレフィックス）を使用し、簡潔な日本語で記述してください。

- `feat:` 新機能の追加
- `fix:` バグの修正
- `docs:` ドキュメントのみの変更
- `style:` コードの動作に影響を与えない変更（ホワイトスペース、フォーマット、セミコロンの追加など）
- `refactor:` バグ修正や機能追加を含まないコードの変更
- `test:` テストコードの追加・修正
- `chore:` ビルドプロセスやツール、ライブラリの変更

## 実行例

- `feat: ユーザーログイン機能の追加`
- `fix: バグの修正（メールバリデーション）`
